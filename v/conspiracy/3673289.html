qwop 2020-02-24T23:00

The glitches only happen around the objects they are manipulating

That is how MPEG works. It only transmits data on things that move in the frame. It does not touch any other pixels. That is part of the invention on how it is able to compress the image.

So wherever you have motion in the image, you will have data being transmitted for that motion. If there is gap in that data, then the motion will glitch. This is an unavoidable result of the MPEG algorithm.

The likelihood the video will glitch anywhere else in the frame, if there is no motion there, is lower, because if there is no motion, there will be no (or less) data transmitted for those blocks.
