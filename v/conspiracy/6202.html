ProgressiveRevert 2014-06-21T11:35

It's a way to encrypt the communication via SSL (In turn, HTTPS when HTTP is piped through SSL layer.) Think of it as a way to prevent eavedropping by securely exchanging the key between the server and client (Look up Diffie-Hellman visual example to grasp the concept how it securely exchange the key.)

Finally, it use encryption like AES or Serpent to encrypt the communication using the same key that secure key exchange created. (Fun fact, there are more key space in Serpent/AES256 than there are atoms on Earth.) Though there are rumors about XAES which is assumed to be 1024 bits encryption, it probably not necessary anytime in the future considering the size of available keyspace.
